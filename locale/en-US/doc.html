Fat-Free Fox

You are viewing this page either because it was opened from add-on preferences or because this add-on has just been installed/upgraded. This page explains some of the probable reasons for altering Firefox features. The objective is to inform and educate the users and not to spread mistrust or panic. This is Fat-Free Fox version 2.2

Search Behavior in Location Bar

In Firefox 43, the location bar was enhanced with extra search related capabilities. When users type something in the location bar, related search suggestions are provided at the end of the dropdown list. Also, Firefox adds a "Visit www.example.com" or "Search with Google" entry at the top of the dropdown list. Users may find these new features annoying and distracting and might want to disable them completely.

Note that for Firefox 42 and below, these features were deactivated by Firefox (this option in the Fat-Free Fox preferences would be selected by default).

Push Notifications by Websites

In Firefox 44, a new feature called Web Push notifications was introduced which allows a website to display notifications to the users even when the site is no longer open. For example, an email provider could display a notification to inform users that a new email has been received. Or, an online retailer could show a notification when an item in user's wish list is about to go out of stock. The system uses servers maintained by Mozilla (Desktop) or Google (Android) using which a site can push notifications to the user's screen even when the site is not open. A basic version of this feature can be seen in action by visiting goroost.com.

Firefox provides options to the users to allow/block these push notifications on a per-site basis. Users can also block notifications temporarily from all sites for the current session of Firefox; when Firefox is restarted, the notifications will be enabled again. However, several users might prefer a quieter browser and might like to disable all such notifications from all websites even after Firefox is restarted.

Note that for Firefox 43 and below, push notifications were deactivated by Firefox (this option in the Fat-Free Fox preferences would be selected by default).

Tracking Protection for non-private Windows

Most of the websites and online services that are perceived as free by the users are not free in the true sense. In exchange for free web content, these sites allow third-party advertisers to harvest users' private data like location, email addresses, areas of interests, likes and dislikes, browsing habits etc. The advertisers use this data to uniquely identify visitors, build a profile and follow them when they surf across the web through multiple websites. These browsing profiles can be very specific and shockingly revealing - mothers with infant children or a person with affiliation to the Democratic Party. This multi-billion dollar digital marketing industry has caused seemingly mundane activities like shopping, chatting and travelling to be constantly recorded, analyzed and monetized.

As long as the user is a willing participant, this sort of tracking can be a good thing. For example, online retailers can tailor their websites to their needs and ensure they are more likely to see advertisements for products and services relevant to their interests; restaurant discovery sites can offer its users an individualized experience based on their location and past preferences. The problem is that most of this information has been secretly harvested without making it clear to the users what is being collected and how that information is being used. A recent report into the US consumers' attitude to this collection of personal data has revealed that most users feel powerless against the ubiquitous "big brother" like advertising networks who are always watching over their shoulders.

Firefox introduced a tracking protection feature in version 42 where it would block third-party content like ads, analytics trackers and social share buttons that may record users' behavior without their knowledge. This built-in tracking protection is similar in nature to the one provided by popular extensions like Adblock Plus and uBlock Origin. However, by default, Firefox enables this feature only for Private Browsing windows. Many privacy-conscious users might wish to benefit from this protection in non-private windows too. Users can also select between Basic and Strict protection in Firefox preferences.

CAUTION: With tracking protection enabled, some web pages may appear broken since ads that track behavior are blocked. Users can disable tracking protection on a per-site basis using the Control Center built into Firefox.

WebRTC Private IP Address Leak

WebRTC (Web Real-Time Communications) is a technology that enables audio/video streaming and data sharing between two browsers without the need for any plugins. Firefox Hello is one such feature built into Firefox that makes use of WebRTC. Real-Time applications like video calls work best when connection is made directly between the two browsers, producing the lowest latency and the best user experience. In order to establish such direct connections, WebRTC collects every IP address that might be used to reach the browser and exposes it to the outside world.

Many users take online privacy and anonymity very seriously and go to great lengths in order to protect their private information like location, IP address etc. Proxies, VPNs and Tor are some of the most common methods used to fight back against browser fingerprinting and mass surveillance. However, it turns out that WebRTC is one of the weak links in their defenses because it exposes users' public and private IP addresses to websites that they visit (even when WebRTC is not being actively used). Even if a user is not behind a proxy or VPN, he still might find it icky to have a private thing exposed to all websites.

Note that enabling this option will only block private IP address leak due to WebRTC. It will not disable WebRTC completely. Firefox Hello would still be usable as before.

Beacon Data

When a user is about to leave a website (by closing the current tab or by navigating to another website), the webpage can send some arbitrary beacon data to the website server. This data is generally used for analytics and diagnostics by the website. Since the user has no control over what information is sent as part of this data, it is open to abuse by website developers and can be used to track and gather information about the users.

Proactive Download of Webpage Content

When a user visits a webpage, the creator of the page can anticipate what links the user will click on next. Based on this anticipation, the webpage can provide a list of web content (html pages, images etc.) to Firefox so that it can be downloaded in the background even before the user clicks those links. When Firefox is idle, it will silently download such content and store it in its cache. If the user actually clicks on those links, they can be loaded quickly from the cache. This technique can be useful in speeding up webpage load times - for example, if the next page has several large images.

However, this look-ahead mechanism can also lead to privacy and security issues because Firefox could download webpages, images and cookies without the knowledge of the user, from sites that the user might never visit. If the user has volume based Internet data plans, he might end up paying for downloads that he neither asked for nor used. It is possible for a rogue website to abuse this feature and silently download malware onto the user's system (drive-by downloads). Also, users may unknowingly violate Internet policies of their organization if such proactive downloads access unauthorized content.

Proactive DNS Lookups

When a user loads a website, Firefox contacts a Domain Name Server (DNS) to map the website's name (say, www.example.com) to its IP address (say, 93.184.216.119). This is called a DNS lookup or resolution. Using this IP address, Firefox establishes a network connection to the website and loads its pages.

In order to make page load time appear faster, Firefox uses a proactive DNS lookup mechanism. While the user is currently browsing a webpage and that webpage has links pointing to another website (say, www.google.com), Firefox speculates that the user will click on one of those links and performs DNS lookup for all of them in advance. While the practical benefits of such speculative performance improvements might be miniscule, experts agree that such practices could lead to privacy loopholes that are ripe for abuse (PDF).

Speculative Network Connections

In certain scenarios, Firefox uses speculative connection mechanism to improve loading time of web sites. For example, when a user hovers the mouse over thumbnails in the New tab page, Firefox opens a network connection to the website server to prepare so that if the user actually clicks the link, the overall speed of loading the web page is faster. Another scenario where Firefox does the same is when the user starts to search in the Search Bar, or in the search field on the Home or the New Tab Page. This predictive connection sends only the IP address of the user to the website server. No request is sent to load the page. No other user-specific data (email address, password etc.) is sent.

It has been reported (and also independently confirmed by the author of this add-on) that in reality speculative connections are not limited to the few scenarios mentioned above. When a user hovers mouse over any normal link of a website, Firefox seems to open speculative connections for those links as well. Mozilla documentation for this feature does not mention anything about normal links also being used for speculative connections. Moreover, Mozilla has not provided any concrete data that quantifies how much performance improvement is actually observed with predictive connections. These comments indicate that such comparative tests were never carried out.

While many users may be comfortable with their IP address being shared with web sites they never intend to visit, a number of users have voiced their concern that this is a violation of their privacy. Normally, users hover their mouse over a link to know where the link actually points to and then decide whether it would be safe to click on the link or not. With speculative connections enabled, Firefox silently communicates with the web site server and shares the users' IP address. This behavior can potentially allow email spammers to track their victims, confirm that their email addresses are active and send them even more spam. Many users feel that such privacy violations are a huge price to pay for non-quantifiable and speculative performance improvements.

Pocket™ for Firefox

Pocket for Firefox is a feature that allows users to save web pages with just a click and read them later. While several users find this feature useful, others feel that it is an unwanted and unnecessary addition to Firefox due to the following reasons.

Privacy Implications

Pocket is a proprietary, third-party service integrated into Firefox. When users sign in to Pocket, their email address is implicitly shared with the Pocket company. When users save web pages for later use, their likes and areas of interest are stored in the servers of the Pocket company. It is quite plausible that this personal information and browsing habits are sold or transferred by Pocket to another company in future. Many users may not fully understand where their personal data is going because they implicitly trust Mozilla and the Pocket feature is bundled with the browser. When Mozilla released Pocket, they failed to inform and educate the users that Pocket is not a Mozilla service but is owned by a third-party whose privacy policy is different from Mozilla's. This is a significant departure from Mozilla's commitment to openness and transparency. Many privacy-sensitive users feel that quietly adding a proprietary service to an open source software like Firefox is against the spirit of a free, open and neutral web.

UI Clutter

Firefox UI is lazy loaded which means that the new Pocket feature has no impact on resource or memory usage until it is actually used. Despite that, several users are simply annoyed by the extra UI clutter because they suddenly noticed this extra thing in their toolbar and Bookmarks menu. They feel that it would be more appropriate if the Pocket feature is available as an optional add-on instead of being directly integrated with the browser.

Firefox Hello

Firefox Hello is a communication system built into Firefox that allows video conversations between users. Many users find this feature useful but several others feel that a video calling service like Hello is not an essential component of browsing experience and might want to get rid of the extra UI clutter.

Reader View

The Reader View feature improves readability of web pages by removing clutter like buttons, ads, background images etc. In case a user does not find it useful, the feature can be disabled which will remove its icon from the address bar.
