Fat-Free Firefox

You are viewing this page either because it was opened from add-on preferences or because this add-on has just been installed/upgraded. This page explains some of the probable reasons for disabling Firefox features. The objective is to inform and educate the users and not to spread mistrust or panic. This is Fat-Free Firefox version 1.0

Speculative Connections

In certain scenarios, Firefox uses speculative connection mechanism to improve loading time of web sites. For example, when a user hovers the mouse over thumbnails in the New tab page, Firefox opens a network connection to the website server to prepare so that if the user actually clicks the link the overall speed of loading the web page is faster. Other scenarios where Firefox does the same is when the user starts to search in the Search Bar, or in the search field on the Home or the New Tab Page. This predictive connection sends only the IP address of the user to the website server. No request is sent to load the page. No other user-specific data (email address, password etc.) is sent.

It has been reported (and also independently confirmed by the author of this add-on) that in reality speculative connections are not limited to the few scenarios mentioned above. When a user hovers mouse over any normal link of a website, Firefox seems to open speculative connections for those links as well. Mozilla documentation for this feature does not mention anything about normal links also being used for speculative connections. Moreover, Mozilla has not provided any concrete data that quantifies how much performance improvement is actually observed with predictive connections. These comments indicate that such comparative tests were never carried out.

While many users may be comfortable with their IP address being shared with web sites they never intend to visit, a number of users have voiced their concern that this is a violation of their privacy. Normally, users hover their mouse over a link to know where the link actually points to and then decide whether it would be safe to click on the link or not. With speculative connections enabled, Firefox silently communicates with the web site server and shares the users' IP address. This behavior can potentially allow email spammers to track their victims, confirm that their email addresses are active and send them even more spam. Many users feel that such privacy violations are a huge price to pay for non-quantifiable and speculative performance improvements.

Pocket™ for Firefox

Pocket for Firefox is a feature that allows users to save web pages with just a click and read them later. While several users find this feature useful, others feel that it is an unwanted and unnecessary addition to Firefox due to the following reasons.

Privacy Implications

Pocket is a proprietary, third-party service integrated into Firefox. When users sign in to Pocket, their email address is implicitly shared with the Pocket company. When users save web pages for later use, their likes and areas of interest are stored in the servers of the Pocket company. It is quite plausible that this personal information and browsing habits are sold/transferred by Pocket to another company in future. Many users may not fully understand where their personal data is going because they implicitly trust Mozilla and the Pocket feature is bundled with the browser. When Mozilla released Pocket, they failed to inform and educate the users that Pocket is not a Mozilla service but is owned by a third-party whose privacy policy is different from Mozilla's. This is a significant departure from Mozilla's commitment to openness and transparency. Many privacy-sensitive users feel that quietly adding a proprietary service to an open source software like Firefox is against the spirit of a free, open and neutral web.

UI Clutter

Firefox UI is lazy loaded which means that the new Pocket feature has no impact on resource or memory usage until it is actually used. Despite that, several users are simply annoyed by the extra UI clutter because they suddenly noticed this extra thing in their toolbar and Bookmarks menu. They feel that it would be more appropriate if the Pocket feature is available as an optional add-on instead of being directly integrated with the browser.

Firefox Hello

Firefox Hello is a communication system built into Firefox that allows video conversations between users. Many users find this feature useful but several others feel that a video calling service like Hello is not an essential component of browsing experience and might want to get rid of the extra UI clutter.

Reader View

The Reader View feature improves readability of web pages by removing clutter like buttons, ads, background images etc. In case a user does not find it useful, the feature can be disabled which will remove its icon from the address bar.
